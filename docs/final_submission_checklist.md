## Report Package

- [ ] Final report is complete in English
- [ ] Required title and declaration pages are present
- [ ] All figures and tables are readable and referenced
- [ ] References and citations are complete
- [ ] Results match repository datasets and logs
- [ ] Limitations and safety boundaries are explicit

- [ ] Final report source is backed up

## Repository

- [ ] Root README reflects the final system and setup
- [ ] Installation and reproduction steps are verified
- [ ] Firmware, gateway, experiments, and documentation are organized
- [ ] Raw data and processing provenance are preserved
- [ ] Automated tests pass or failures are documented
- [ ] No credentials, tokens, private keys, or personal secrets are tracked
- [ ] Generated files and local environments are ignored appropriately
- [ ] Repository links and internal paths work
- [ ] Final revision is committed and pushed
- [ ] Git working tree is clean
- [ ] Repository remains shared with `matusformanek`
- [ ] Final revision identifier or tag is recorded

## Presentation

- [ ] Slides are complete in English
- [ ] Problem, architecture, method, results, safety, limitations, and conclusion are covered
- [ ] Text and figures are readable at presentation distance
- [ ] Claims match final report results
- [ ] Presentation fits the required time
- [ ] Questions and backup slides are prepared
- [ ] Presentation source and exported copy are backed up

## Demonstration

- [ ] Required hardware and cables are available
- [ ] ESP32-C6 firmware and laptop environment are verified
- [ ] Local runtime and selected model are available offline
- [ ] Demonstration starts from a documented clean state
- [ ] Normal, anomaly, passive, validation, and safe-fallback paths are prepared
- [ ] Active mode is disabled by default
- [ ] Safe shutdown is rehearsed
- [ ] Recorded video, screenshots, logs, or plots are available as fallback
- [ ] Demonstration duration is appropriate

## Reproducibility and Evidence

- [ ] Environment and version information is recorded
- [ ] Experiment configurations and sample counts are documented
- [ ] Raw datasets are unchanged and accessible
- [ ] Processed metrics can be reproduced
- [ ] Prompt, schema, whitelist, and parser versions are identified
- [ ] Negative and failed results are retained where relevant
- [ ] Report tables and figures can be traced to evidence

## Backups

- [ ] Repository exists locally and on GitHub
- [ ] Report and presentation exist in at least two independent locations
- [ ] Raw experiment data has an additional backup
- [ ] Exported PDF and presentation open from the backup
- [ ] Recovery steps are known if the demonstration laptop fails
